Terminal Multiplexers

1.
ssh로 서버에 연결하여 작업할 때 단말기가 하나 밖에 없지만 여러대를 가진 것과 같은 효과를 원할 때
2.
실행시간이 긴 프로그램을 돌려놓고 다른 일을 하고 싶을 때
3.
실행시간이 긴 프로그램을 실행할 때 원격 연결은 끊고 서버에서 계속 실행하고 싶을 때
4.
하나의 윈도우에서 여러개의 터미널 세션을 다루고자 할 때 유용
2,3 번은 nohup 명령어를 사용할 수 있지만 tmux
4번은 새로운 터미널 창을 열면 해결되는 문제이지만, terminal multiplexer가 훨씬 더 유연한 해결책이다!
